# MCP Terminal Bridge

A lightweight OAuth2-protected bridge to expose a local terminal (or any shell) as an MCP (Model Context Protocol) endpoint. This allows AI agents or other tools to execute commands on your machine securely.

## Features

- OAuth2 authorization flow for secure access
- Dynamic client registration
- Cross-platform support (Windows & Kali Linux)
- Auto-generated master key for authentication
- Cloudflare Tunnel integration for public access

## Use Cases

- Secure remote command execution for AI agents
- Terminal access via OAuth2-protected API
- Integration with MCP-compatible tools

## Prerequisites

- Node.js (v16+ recommended)
- Cloudflared (for tunneling)
- Git (for cloning)

## Installation

### Clone the Repository

```bash
git clone https://github.com/jojin1709/MCP-Terminal-Bridge.git
cd MCP-Terminal-Bridge
```

### Install Dependencies

```bash
npm install
```

### Install Cloudflared

Download and install Cloudflared for your platform:

- **Linux (Kali)**:
  ```bash
  wget https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb
  sudo dpkg -i cloudflared-linux-amd64.deb
  ```
- **Windows**:
  Download the executable from [Cloudflare's releases page](https://github.com/cloudflare/cloudflared/releases/latest) and place it in the project directory.

## Running the Server

### Kali Linux

1. Make the start script executable:
   ```bash
   chmod +x start.sh
   ```
2. Run the server:
   ```bash
   ./start.sh
   ```

### Windows

1. Run the PowerShell script:
   ```powershell
   .\start.ps1
   ```

## Configuration

The server uses environment variables for configuration:

- `PUBLIC_URL`: Your tunnel URL (auto-set by start scripts)
- `MASTER_KEY`: Authentication key (auto-generated by start scripts)

## API Endpoints

- `GET /.well-known/oauth-authorization-server`: OAuth2 metadata
- `POST /register`: Dynamic client registration
- `GET /authorize`: Authorization endpoint
- `POST /token`: Token exchange
- `POST /mcp`: MCP endpoint (protected)

## Example Usage

### Register a Client

```bash
curl -X POST http://localhost:8787/register \
  -H "Content-Type: application/json" \
  -d '{"redirect_uris": ["http://localhost:3000/callback"]}'
```

### Get OAuth2 Metadata

```bash
curl http://localhost:8787/.well-known/oauth-authorization-server
```

## Troubleshooting

### Port Already in Use

If you encounter an error like `Error: listen EADDRINUSE: address already in use :8787`, it means the port is already occupied. You can either:

- Kill the process using the port:
  ```bash
  sudo lsof -i :8787
  kill -9 <PID>
  ```
- Change the port in `server.js` and update the `start.sh`/`start.ps1` scripts accordingly.

### Cloudflared Not Found

Ensure `cloudflared` is installed and available in your system's PATH. For Windows, place `cloudflared.exe` in the project directory.

### Missing Dependencies

Run `npm install` to install all required Node.js dependencies.

## Security Notes

- Never share your MASTER_KEY
- The server uses in-memory storage (not for production)
- Always use HTTPS for public exposure

## Environment Variables

You can manually set the environment variables if you prefer not to use the start scripts:

### Kali Linux

```bash
export PUBLIC_URL=https://your-tunnel-domain
export MASTER_KEY=your-secret-key
node server.js
```

### Windows

```powershell
$env:PUBLIC_URL="https://your-tunnel-domain"
$env:MASTER_KEY="your-secret-key"
node server.js
```

## FAQ

### What is MCP?

MCP (Model Context Protocol) is a protocol for exposing tools and endpoints to AI agents or other tools.

### Can I use this in production?

This project is designed for personal or development use. For production, consider using a database for storing clients, tokens, and codes.

### How do I secure my server?

- Always use HTTPS.
- Keep your MASTER_KEY secret.
- Restrict access to trusted clients only.
